# 方案 B：使用 GitHub Actions 云编译 APK

**优势：**
- ✓ 完全免费
- ✓ 不需要安装任何工具
- ✓ 在云服务器编译，不占用本地资源
- ✓ 编译速度快（5-20 分钟）
- ✓ 可以自动编译多个版本

## 快速开始（3 步）

### 第 1 步：上传代码到 GitHub

1. **创建 GitHub 账号**（如果没有的话）
   - 访问 https://github.com
   - 点击 Sign up，注册账号

2. **创建新仓库**
   - 登录后点击右上角 `+` → New repository
   - 仓库名：`divination` 或 `python-divination`
   - 选择 `Public`（必须是公开的，因为免费账户要求）
   - 点击 Create repository

3. **上传你的项目文件**
   
   **方式 1：使用 Git 命令（推荐）**
   ```powershell
   cd c:\Users\黎呈君\Desktop\python_code
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/你的用户名/divination.git
   git push -u origin main
   ```
   
   **方式 2：网页上传**
   - 在 GitHub 仓库页面点击 Add file → Upload files
   - 选择你本地项目文件夹中的所有文件
   - 上传完成

### 第 2 步：GitHub Actions 会自动编译

1. **推送代码后自动触发**
   - 上传完代码，GitHub 会自动启动编译工作流
   
2. **查看编译进度**
   - 点击仓库中的 `Actions` 标签
   - 找到 "Build Android APK" 工作流
   - 查看编译进度（实时日志）

3. **等待编译完成**
   - 编译通常需要 5-20 分钟
   - 工作流右侧会显示 ✓ 或 ✗

### 第 3 步：下载编译好的 APK

1. **下载 APK 文件**
   - 编译完成后，点击工作流进入详情
   - 向下滚动找到 "Artifacts" 部分
   - 点击 `divination-apk` 下载 zip 文件
   - 解压后得到 APK 文件

2. **APK 文件位置**
   ```
   divination-apk/
   └── bin/
       ├── divination-0.1-debug.apk
       └── ...其他编译文件
   ```

3. **传到手机安装**
   - 将 APK 复制到手机
   - 或用 USB 连接：`adb install divination-0.1-debug.apk`

## 文件结构检查

确保你的 GitHub 仓库包含以下文件：

```
divination/
├── .github/
│   └── workflows/
│       └── build-apk.yml          ← GitHub Actions 配置（已创建）
├── buildozer.spec                  ← 编译配置（已有）
├── requirements.txt                ← Python 依赖（已有）
├── divination_app.py               ← 主程序（已有）

```

## GitHub 高级用法

### 自动发布版本

修改工作流文件，添加发布功能：
1. 编辑 `.github/workflows/build-apk.yml`
2. 添加 Release 步骤
3. 自动创建 Release，包含 APK 下载链接

### 手动触发编译

不需要 push 代码也能触发编译：
1. 在 Actions 标签页
2. 点击 "Build Android APK"
3. 右侧有 "Run workflow" 按钮
4. 点击即可手动启动编译

### 多个编译版本

GitHub Actions 可以同时编译多个版本（如不同架构、不同签名等）

## 常见问题

### Q: GitHub Actions 有限额吗？
**A**: 免费账户每月有 2000 分钟的免费额度，这个项目每次编译约 10 分钟，足够月编译 200 次了。

### Q: APK 下载后多久会过期？
**A**: GitHub 保存 30 天，之后自动删除。如需长期保存，下载后自己保存。

### Q: 如何编译发布版本（带签名）？
**A**: 
1. 生成签名密钥：
   ```bash
   keytool -genkey -v -keystore my-release-key.keystore -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias
   ```
2. 在 GitHub Settings → Secrets 中添加密钥信息
3. 修改工作流配置使用密钥

### Q: 编译失败了怎么办？
**A**: 
1. 查看 Actions 中的编译日志
2. 常见错误：
   - 缺少依赖：检查 requirements.txt
   - Python 版本问题：修改工作流中的 Python 版本
   - buildozer.spec 配置问题：检查配置文件

## 下一步

现在项目已准备好通过 GitHub Actions 编译，按以下步骤操作：

1. 安装 Git（如果还没有）

2. 上传代码到 GitHub（选择方式 1 或 2）

3. 等待自动编译完成

4. 下载 APK 安装到手机
